## 2024-05-24 - Prevent Path Traversal in Filename Construction
**Vulnerability:** Combining an unvalidated, user-provided filename directly with an output directory path using `Path.Combine` allows attackers to supply inputs containing traversal sequences (e.g., `../../malicious`), enabling them to escape the intended directory and potentially overwrite critical system or application files elsewhere.
**Learning:** Functions designed to handle user-specified filenames often falsely assume the input only represents the name itself, overlooking the fact that file APIs may interpret directory separators, thereby inadvertently accepting a relative path payload.
**Prevention:** Always extract only the filename component from untrusted input using robust standard library functions such as `System.IO.Path.GetFileName()`. Additionally, guard against the edge case where the extracted result is entirely empty (which occurs if the malicious input consisted only of traversal characters) by supplying a safe, hardcoded default filename (e.g., `output.vsdx`).
