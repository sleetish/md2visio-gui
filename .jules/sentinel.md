## 2025-03-08 - Path Traversal via Incomplete Filename Sanitization
**Vulnerability:** Filename extraction methods like `Path.GetFileName` and `Path.GetFileNameWithoutExtension` are vulnerable to path traversal (CWE-22) when they are not aware of mixed or cross-platform slashes (e.g. `\` vs `/`). A user could supply an input path like `..\..\..\etc\passwd` which, if only standard OS separators are considered, could bypass simple extraction and write to arbitrary directories.
**Learning:** Depending on the execution environment, `Path.GetFileName` might not strip out slashes belonging to other OS platforms. Normalizing slashes across platforms before extracting the filename is required to ensure that the file is safely constrained.
**Prevention:** Always normalize slashes by replacing both forward slashes (`/`) and backward slashes (`\`) with `Path.DirectorySeparatorChar` before calling `Path.GetFileName` or `Path.GetFileNameWithoutExtension`. Additionally, ensure that if the sanitized file name results in an empty string, a safe fallback string (like `"output"`) is assigned to prevent downstream errors.
