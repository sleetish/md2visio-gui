## 2024-05-18 - [Path Traversal in Output Filename Construction]
**Vulnerability:** The application's GUI allows users to specify an output file name (`fileName`), which is directly combined with the output directory path (`outputDir`) via `Path.Combine` without sanitization. This allows path traversal attacks (e.g., using `../../../etc/passwd` as the filename) to write files to arbitrary locations on the system.
**Learning:** `Path.Combine` handles absolute paths and directory traversal sequences by resolving them relative to the first argument or using the absolute path provided in the second argument. Simply appending an extension does not mitigate the traversal.
**Prevention:** Always sanitize user-provided filename components using `System.IO.Path.GetFileName()` to extract just the file name and strip any directory traversal segments. Always handle the edge case where the sanitized result is empty (e.g., if the input was solely traversal characters) by providing a safe default string (e.g., "output").
