## 2024-03-17 - Path Traversal Vulnerability in BuildOutputPath
**Vulnerability:** The `BuildOutputPath` function in `ConversionService` did not sanitize the user-provided `fileName` parameter before using `Path.Combine` with the `outputDir`. This meant that an attacker could provide a malicious `fileName` like `../../etc/passwd` or `..\..\Windows\System32\cmd.exe` to write arbitrary files to restricted locations.
**Learning:** `Path.Combine` and path-building logic are highly susceptible to directory traversal when concatenating a base directory with an unsanitized path segment, particularly when handling edge cases around absolute paths and trailing slashes.
**Prevention:** Always sanitize components of user-provided filenames. Normalize cross-platform slashes (e.g., `.Replace('/', Path.DirectorySeparatorChar).Replace('\', Path.DirectorySeparatorChar)`) before applying `System.IO.Path.GetFileName()`, as this method alone may be insufficient for mixed slashes. Always handle the edge case where the sanitized result is empty by providing a safe default string (e.g., 'output' or 'default').
