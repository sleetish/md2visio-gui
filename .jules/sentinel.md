
## 2024-05-27 - [Fix Path Traversal in ConversionService]
**Vulnerability:** The `ConversionService.BuildOutputPath` method combined the `outputDir` directly with the user-provided `fileName` using `Path.Combine()`. A malicious user could potentially input a `fileName` containing directory traversal sequences (like `..\..\..\some\path`), leading to arbitrary file write outside of the intended `outputDir`.
**Learning:** `Path.Combine` doesn't sanitize the inputs it combines, and if the second argument is an absolute path or contains traversal characters, it can escape the intended root. Additionally, just relying on `Path.GetFileName` isn't always foolproof on cross-platform scenarios if the slashes are mixed (`/` and `\`), so it's important to normalize slashes and fallback to a default safe name if the resulting file name is empty.
**Prevention:** Always normalize slashes (`.Replace('/', Path.DirectorySeparatorChar).Replace('\', Path.DirectorySeparatorChar)`) and apply `Path.GetFileName()` on user-supplied filename components before combining them with a directory path. Provide a safe fallback (e.g., `"output"`) if the sanitized filename evaluates to null/whitespace.
