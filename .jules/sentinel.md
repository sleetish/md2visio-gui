## 2025-02-17 - [CRITICAL] Fix DoS via Stack Overflow in custom JSON parsers
**Vulnerability:** A Denial of Service (DoS) vulnerability existed in `md2visio/struc/figure/MmdJsonObj.cs` and `md2visio/struc/figure/MmdJsonArray.cs`. These files implement custom recursive JSON parsers (`Load` methods) that recursively instantiate themselves and traverse nested objects and arrays without enforcing limits on the maximum recursion depth. An attacker providing deeply nested JSON (e.g., `{{{{...}}}}` or `[[[[...]]]]`) could cause a StackOverflowException leading to a crash.
**Learning:** Custom parsing logic involving recursion must account for malicious, intentionally deep inputs. While testing, it was identified that unclosed and empty recursive JSON structures (like continuous arrays `[[[[...`) trigger syntax checks, but fully unclosed arrays with syntax valid nested types can easily exhaust stack memory via unchecked constructor recursion.
**Prevention:** Enforce a maximum recursive depth limit (`MAX_DEPTH`) across mutually recursive or self-recursive parsing pathways, incrementing the tracking metric correctly through all branches, and throwing an explicit exception (`InvalidOperationException`) to fail securely before exceeding the stack size limit.
